Filter InfoFilter KindHistory plug-in

The History plug-in is displayed when you choose History in the Filter Kind pop-up menu in the Filter Info pane.

This plug-in is used by the default Change Only If Unique and History Always Repeats filters.

This plug-in enables detection of a previous occurrence of the input text. When a filter based on this plug-in is used in a test, the test keeps track of the input text from each check, and this filter can change the result status of the check based on whether the text has been seen before or not.

It has two pop-up menus: one for the result to use if the text is unique, and one for not unique. For each, the available options are Auto-Detect, to use the normal rules on change detection, Unchanged to leave the status as it was from a previous filter, Changed to make the status indicate a change occurred, or Failure to make the status indicate an error.

The built-in History-based filters demonstrate ways this can be used, but there are many other combinations.

This filter outputs the input text.
