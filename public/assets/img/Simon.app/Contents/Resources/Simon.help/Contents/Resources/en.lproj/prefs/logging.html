PreferencesLogging

The Preferences window is displayed via the SimonPreferences... menu item. The Logging page includes options related to logging information about Tests.

Maximum number of checks to remember per test: This field specifies how many checks to store for each test. After this number is reached, the oldest entries are removed so that only this number remain. Note that this value is per test, so when displaying checks for a combination of tests, the number displayed may be greater. This defaults to 500.

Maximum number of changes to remember per test: This field specifies how many changes to store for each test. This defaults to 100.

Maximum number of failures to remember per test: Similar to the previous field, this one specifies the maximum number of failures to store for each test. This also defaults to 100.

Maximum number of activities to remember per test: Also similar to the previous fields, this one specifies the maximum number of activities to store for each test. This defaults to 200.

Display change differences without HTML tags: If this is checked, the changes log filters out the HTML tags that may be present (e.g. "<p>"), so you can concentrate on the relevant values as they appear on a rendered page. By default this is off, so HTML tags are included.

Restore Defaults: Click this button to change the preferences on this page back to the factory-set values. The default values usually represent the recommended configuration, though of course preferences are there to be changed if desired. Preferences on other pages are not affected.
