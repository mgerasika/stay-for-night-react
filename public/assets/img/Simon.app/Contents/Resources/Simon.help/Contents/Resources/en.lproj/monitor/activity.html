MonitorActivity

The Activity log at the bottom of the Monitor window contains a table listing any activity for the tests. This can include when tests are edited, paused, resumed, started checking, stopped, completed, plus the status and output of the service and any filters, along with notifications. As with the changes log, if no tests are selected, the activity from all tests are listed; if some or one are selected, only the activity from those selected tests are listed. The columns can be sorted, and the order toggled, as desired.

Also included is a search field. This is useful to refine the information shown in the log even further. You can select one or more tests in the tests table, then enter some text in the search field to view just the log lines that contain that text.

Here are the details of the columns:

Activity Date: This column displays the date and time that the action occurred.

Test Name: This column displays the name of the test that this relates to; useful when displaying activity for multiple tests.

Type: This column (and the icon column preceding it) displays the type of activity: Service, Filter, Test editing, etc.

Kind: This column indicates what kind of item was used (i.e. the plugin kind).

Kind Name: This column displays the name of the item that was used (service, filter, or notifier).

Status: This column contains a status triangle indicating status of the activity: upwards-facing black for unchanged upwards-facing green for changed, downwards-facing red for failures, and upwards-facing orange for recoveries.

Details: This column displays additional information, such as the output of the service or filter.
