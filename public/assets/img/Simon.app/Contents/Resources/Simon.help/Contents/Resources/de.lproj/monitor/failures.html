MonitorFehlschläge

Das Fehlschläge-Protokoll am unteren Rand des Simon-Monitor-Fensters enthält eine Tabelle, die die letzten Fehlschläge der Tests zeigt. Wie bei den Änderungen gilt auch hier: Wenn keine Tests selektiert sind, werden die Fehlschläge aller Tests gelistet; wenn einige Tests oder nur ein Test selektiert sind, werden nur die Fehlschläge der selektierten Tests gelistet. Die Spalten der Tabelle können sortiert und waagerecht verschoben werden, falls gewünscht.

Auch dieser Bereich bietet ein Suchfeld. Nutzen Sie es, um in der Tabelle gezeigte Informationen weiter einzugrenzen. Selektieren Sie z.B. einen Test oder mehrere Tests in der Tests-Tabelle, geben Sie dann einen Text im Suchfeld ein, und Sie werden nur die die Tabellen-Zeilen sehen, die diesen Text enthalten.

Hier sind die Details zu den einzelnen Spalten:

Fehlgeschlagen:
Diese Spalte zeigt, wann die letzte Prüfung fehlgeschlagen ist, oder genauer gesagt, wann Simon die Website geprüft und einen Fehler irgendeiner Art empfangen hat. Eine Website kann also theoretisch durchaus schon ein paar Minuten früher ausgefallen sein.
Gerettet:
Diese Spalte zeigt, wann der Test nach Fehlschlägen das letzte Mal wieder erfolgreich geprüft wurde, oder "noch nicht", falls der Fehlschlag noch nicht gerettet worden ist. Eine Rettung liegt z.B. dann vor, wenn eine Website nach einem Fehlschlag und einer Folge von weiterführenden Prüfungen wieder erreichbar ist.
Zeit:
Diese Spalte zeigt die ungefähre Zeit, die zwischen dem Fehlschlag und der Behebung vergangen ist, so dass Sie auf einen Blick sehen können, wie lange z.B. eine Website nicht erreichbar war.
Test-Name:
Diese Spalte zeigt den Namen des Tests, dessen Fehlschläge hier aufgeführt werden; nützlich, um mehrere Tests unterscheiden zu können.
Fehler:
Diese Spalte zeigt die Fehlermeldung, die vom Remote-Server oder vom Betriebssystem empfangen oder von Simon generiert wurde. Normalerweise lassen sich daraus Rückschlüsse auf die Ursache des Fehlschlags ableiten.
